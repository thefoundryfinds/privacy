Privacy Policy

Effective date: July 16, 2026

This privacy policy describes how The Foundry Finds ("we", "us", "our") handles data in connection with our internal application that integrates with the Pinterest API ("the App"). The App is used solely to manage The Foundry Finds' own Pinterest account and is not distributed to or used by any other individuals or businesses.

What the App does

The App connects to Pinterest via Pinterest's official OAuth 2.0 authorization flow to read and manage boards and Pins belonging to The Foundry Finds' own authenticated Pinterest account. This includes actions such as creating Pins, creating or reading boards, and reading basic account information for that same account.

Data we access

The App only accesses data belonging to the single Pinterest account that explicitly authorizes it through Pinterest's consent screen. The App does not access, collect, or process data belonging to any other Pinterest user, and does not offer sign-in or data access to third parties.

Data we store

The App stores only the OAuth access and refresh tokens issued by Pinterest, which are required to keep the connection authorized. These tokens are kept in environment variables / a secrets manager on infrastructure controlled by The Foundry Finds and are never committed to source control, logged, or shared with any third party. We do not store Pinterest account passwords; the App never has access to them, as authentication is handled entirely by Pinterest's own login and consent pages.

Data sharing

We do not sell, rent, or share any data obtained through the Pinterest API with third parties. Data is used exclusively to operate The Foundry Finds' own Pinterest presence.

Data retention and revocation

Stored tokens are retained only for as long as the App is in active use. Access can be revoked at any time from the Pinterest account's connected-apps settings, which immediately invalidates the stored tokens.

Security

Access to stored tokens is restricted to authorized personnel of The Foundry Finds. No credentials are stored in source code or shared in plaintext outside of secured configuration.

Changes to this policy

We may update this policy from time to time. Changes will be posted on this page with an updated effective date.
